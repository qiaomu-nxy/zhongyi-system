# 中医问诊管理系统

面向中国大陆中医馆的小团队问诊管理系统。

V1 目标是先快速验证业务流程，采用单诊所单医师模式：
- 患者端：H5 网页
- 医师端：Web 响应式后台
- 后端：FastAPI
- 数据库：SQLite
- 部署：Render 免费云平台

## V1 核心能力

- 患者扫码后登录、预约、当天签到、填写结构化症状
- 医师账号密码登录，查看今日就诊、撰写病历、管理预约与排班
- V1 按单诊所单医师运行，保留 `admin` + `doctor` 两种角色
- 支持治疗过程图表分析
- 支持数据备份导出

## 角色与权限

- `doctor`：V1 默认唯一接诊医师，可查看本诊所患者、预约、排班
- `admin`：负责初始化与管理，可重置默认医师密码、合并患者档案、修改已提交病历

## 患者端规则

- V1 登录方式：`手机号 + 姓名`
- 不支持代家人预约/代填
- 患者识别按“手机号 + 姓名”
- 若患者更换手机号，先按新患者处理，后续由 `admin` 视情况合并
- 患者只能查看自己的历史摘要和简短医嘱
- 患者不能查看完整病历、辨证思路、处方全文、医师备注

## 预约与接诊规则

- V1 为单医师模式，患者预约时无需选择医师
- 同一患者同一时间只能有一个有效预约
- 可预约未来 7 天内时段
- 当天剩余时段可在截止时间前预约
- 改预约只支持“取消后重约”
- 患者可在就诊前 1 天取消预约，当天不可取消
- 患者取消后，时段立即重新开放
- 医师可随时取消预约
- 医师取消后，患者可再次预约当天其他剩余时段
- 患者到店后先签到，再进入待接诊队列
- 同一患者同一天只能有一条就诊记录
- 未到诊患者由医师在营业结束后标记为“爽约”
- V1 只记录爽约次数，不做限制预约

## 病历与数据规则

- 病历提交后，普通医师不可修改，只有 `admin` 可修改
- 保存病历后，系统自动完成本次就诊与关联预约
- V1 不做真删除，只做取消、停用、作废
- 医师离职或停用前，需要先转移患者和预约，再停用账号
- V1 不支持病历常用模板

## 统计口径

- 就诊次数按整个诊所累计
- 新患者：第一次来本诊所
- 复诊患者：以前来过至少一次
- 沉睡患者：超过 1 个月未复诊

## 技术栈

- 后端：Python + FastAPI + SQLAlchemy 2.0 + Pydantic v2
- 数据库：SQLite
- 患者端：Vue 3 + TypeScript + Vant
- 医师端：Vue 3 + TypeScript + Element Plus
- 图表：ECharts

## 本地启动

当前仓库文档已完善，代码可按以下目标结构开发：

1. 启动后端
   - 进入 `backend/`
   - 安装依赖
   - 启动 FastAPI 服务

2. 启动患者端
   - 进入 `patient-h5/`
   - 安装依赖
   - 运行 Vite 开发服务

3. 启动医师端
   - 进入 `doctor-admin/`
   - 安装依赖
   - 运行 Vite 开发服务

接口与页面开发顺序请以 `DEVELOPMENT_GUIDE.md` 为准。

## 部署说明

### V1

- 部署到 Render 免费云平台
- 后端提供 API
- 患者端和医师端可分别构建后部署，或由 FastAPI 统一托管静态文件
- SQLite 数据库存储为单文件，便于备份和迁移

### V2

- 迁移到正式云服务器
- 患者端从 H5 升级为微信小程序
- 后端接口与数据库结构尽量保持兼容，减少迁移成本
- 升级为单诊所多医师模式：患者可选医师、独立排班、独立预约

## 版本规划

### V1

- 单诊所单医师 + H5 患者端 + Web 医师端 + FastAPI + SQLite
- 目标：快速验证诊所业务流程

### V2

- 单诊所多医师 + 微信小程序患者端 + 云服务器
- 增强能力：结构化处方、操作日志、更多运营与分析能力
